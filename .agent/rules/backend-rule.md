# 必须遵守的规则

1. 后端使用Drogon C++ Web Framework，所有扩展功能的实现方案必须优先符合drogon的架构设计，优先使用drogon提供的功能模块作为解决方案，而不是优先引入三方库
2. 项目需要面向对象的思维高度结构化
3. windows平台下，由于项目使用conan管理依赖项，所以必须使用我提供的build.bat来编译
4. 项目配置文件统一在config.json中管理
5. windows平台下，后端程序运行前，必须检查exe同级目录下是否存在配置文件，否则会造成后端程序启动找不到配置文件而崩溃
6. 测试前，先检查条件是否满足。尤其是配置文件以及数据库数据是否正确等
7. 每个迭代完成后，先同步更新下所有相关文档，然后commit代码
