# 必须遵守的规则

1. 所有最终回复内容必须以中文回复
2. 项目需要面向对象的思维高度结构化
3. 如果方案你不太确定是否正确，不要做，寻找更有把握的方案
4. 后端使用Drogon C++ Web Framework，所有扩展功能的实现必须符合drogon的架构设计，在drogon框架范围外的功能的方案则自己思考。配置文件统一在config.json
5. 由于项目使用conan管理依赖项，所以必须使用我提供的build.bat来编译
6. 后端程序运行前配置文件需要放到exe同级目录
7. 调试代码在问题解决后移除
8. 代码完成后，测试通过才算完成。测试前，先检查条件是否满足。尤其是配置文件以及数据库数据是否正确等
9. 每个迭代完成后，先同步更新下所有相关文档，然后commit代码，不要执行push
